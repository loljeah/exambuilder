## Architecture Notes

### Daemon Communication
- Unix socket at `$XDG_RUNTIME_DIR/kgate.sock`
- Text protocol: `COMMAND args\n` → `OK data\n` or `ERR message\n`
- JSON payloads for complex responses

### Knowledge Tracking
- SM-2 spaced repetition algorithm
- Status: unseen → learning → mastered
- Mastery threshold: 80%

### Logging
- `KGATE_LOG_LEVEL` env var (debug/info/warn/error)
- `KGATE_LOG_JSON=1` for JSON format
- Session ID injected into all log entries
